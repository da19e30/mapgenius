## Summary
The codebase was examined for the OWASP Top 10 risks. Several critical and high‑severity issues were identified, primarily around input validation, insecure defaults, and missing security controls.

## Additional Hardening
- **Rate limiting** on auth endpoints to mitigate brute‑force attacks.
- **Content‑type validation** for uploads; reject non‑allowed MIME types.
- **File sanitisation** – scan uploaded files with ClamAV or similar before processing.
- **Secure cookie flags** (`Secure`, `HttpOnly`, `SameSite=strict`).
- **Enable OpenAPI security schemes** to enforce JWT validation globally.
